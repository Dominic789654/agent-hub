# Code of Conduct

## Our Standard

This project aims to stay technically rigorous, practical, and respectful.

Participants are expected to:

- focus on the work and the evidence
- give actionable, respectful feedback
- assume good faith while still being precise about problems
- keep discussion aligned with the documented local-first OSS scope

Unacceptable behavior includes:

- harassment, insults, or personal attacks
- public sharing of secrets, credentials, or exploit details that put users at risk
- intentionally disruptive behavior in issues, pull requests, or discussions
- pressure to expand the project into unrelated hosted or private infrastructure assumptions

## Enforcement

Maintainers may edit, hide, lock, or remove content that violates these expectations.

Repeated or severe violations may lead to temporary or permanent removal from project participation.

## Reporting

For conduct concerns, contact the maintainers through the repository owner contact path on GitHub. For security-sensitive issues, use the process in `SECURITY.md`.

## Scope

This code of conduct applies across repository issues, pull requests, review comments, and other project-managed collaboration spaces.
