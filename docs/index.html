Local-first · SQLite-backed · OSS MVP

One place to route, observe, and unblock agent work.

agent-hub gives a single operator a concrete control plane for multi-project agent workflows: queue tasks, model dependencies, launch templates and pipelines, track retries and blocked work, surface human handoff, and inspect the entire system through CLI, HTTP, JSON, and a thin dashboard.

Why it exists

Keep orchestration small and inspectable.

The project is intentionally narrow: standard-library Python, local state, explicit queue records, and clear extension points instead of a heavy workflow platform.

CLI + HTTP + UI Operate through scripts, endpoints, or the browser app.
Projects as routing keys Point tasks at local repos and reusable command surfaces.
Human handoff built in Make ambiguity and failures visible instead of hiding them.
Portable by default Everything runs from local files and a small SQLite database.
Good control planes do not hide work. They make task state, dependencies, retries, and operator intervention explicit.

Main capabilities

The current release focuses on the pieces you need before layering on bigger workflow systems: durable queue state, dependency handling, project routing, and thin operator visibility.

Queue core

Durable task lifecycle

SQLite-backed tasks, dependency edges, retries, cancellation, blocked-state propagation, and run history.

Project routing

Actions, templates, and pipelines

Route work into reusable project-backed commands and instantiate bounded multi-step flows.

Operator visibility

Inspect through multiple surfaces

Use CLI commands, JSON endpoints, a minimal HTML page, and the thin browser app at /app.

Handoff policy

Make human intervention first-class

Surface needs_human, notes, and labels so unresolved work does not disappear inside automation.

Saved views

Persist and re-run filtered views

Create saved queries for tasks and pipeline runs, then execute them later from CLI or HTTP.

Public-safe baseline

Small codebase, real extension points

Keep the repo easy to audit while preserving room to extend executors, policies, and integrations.

Where it fits

agent-hub works best when one operator needs one control surface for many local repos and bounded agent tasks.

Cross-repo task routing

Accept natural-language intent, map it to a target project, and launch a bounded task with explicit deliverables.

Serial and parallel workflows

Model dependencies when one task must wait, while still allowing unrelated work to run in parallel.

Human-in-the-loop automation

Run as much as possible automatically, then stop clearly when a hard case needs manual review.

Architecture

The implementation is deliberately plain: small modules, explicit storage, and a dependency graph you can reason about.

static architecture diagram
Implemented slice

Core modules

  • db.py bootstraps local SQLite state
  • repository.py manages lifecycle, edges, and block propagation
  • dispatcher.py claims and executes ready tasks
  • web.py exposes health, status, task, inbox, and dashboard surfaces
  • services/executors.py routes project-backed commands
  • services/pipelines.py expands pipeline templates into task graphs

How work moves

The system keeps the workflow small: define work, dispatch only ready tasks, and surface exceptions for operator action.

1

Define work

Queue a direct task or instantiate a reusable task template or pipeline from the project registry.

2

Dispatch safely

The dispatcher claims only ready tasks and respects dependency completion rules before execution.

3

Observe and intervene

Inspect snapshots, labels, notes, and the human inbox to retry, cancel, or hand work back to a person.

Fastest way to try it

Start the web surface, run the dispatcher, queue example work, then open the dashboard and browser app.

demo commands 
python -m agent_hub serve --port 8080
python -m agent_hub dispatch
python -m agent_hub run-task-template sample-project summarize-input --input "review this idea"
python -m agent_hub run-pipeline sample-project sample-flow --input "demo"
python -m agent_hub list-human-inbox
python -m agent_hub dashboard

What you will see

A queue that processes tasks and pipelines, a human inbox for triage, saved queries, and a read-only dashboard snapshot.

What it is not trying to be

Not a hosted SaaS, not a multi-tenant engine, and not a polished production UI. The point is a small, extensible OSS baseline.

Built for

Clear local orchestration

  • Single-operator or small-team local control planes
  • Explicit routing into many local repositories
  • Bounded agent tasks with visible retries and failures
  • Portable workflows that stay inspectable on disk
Not for

Heavy hosted workflow assumptions

  • Multi-tenant SaaS control planes
  • Opaque remote execution platforms
  • Auth-heavy production service boundaries
  • Enterprise orchestration before core workflow clarity
