Local-first · SQLite-backed · OSS MVP

One control plane for routing and observing agent work across projects.

agent-hub gives you a small, concrete control surface for multi-project agent workflows: queue tasks, model dependencies, launch pipelines and templates, track retries and blockages, surface human handoff, and inspect everything through JSON, CLI, and a thin dashboard.

Main capabilities

The current public release is intentionally narrow: local-first, single-operator oriented, and easy to extend. It focuses on the core building blocks you need before adding heavier workflow systems.

Task queue core

SQLite-backed storage, lifecycle state, dependency edges, retries, cancellation, and blocked-state propagation.

Project routing

Route work into project-backed actions, reusable task templates, and multi-step pipelines.

Human handoff

Surface actionable tasks through needs_human, notes, labels, and the derived human inbox.

Operator visibility

Inspect the system through CLI, JSON endpoints, a classic HTML page, and the thin browser app.

Saved views

Persist query presets for tasks and pipeline runs, then execute them later with one command or route.

Public-safe foundation

Small standard-library codebase, synthetic examples, OSS governance files, and release-oriented docs.

How it works

1

Define work

Queue a task directly, or instantiate a task template / pipeline from a project registry.

2

Dispatch safely

The dispatcher claims only ready tasks and respects dependency completion rules.

3

Observe and intervene

Use dashboard snapshots, pipeline run tracking, notes, labels, and the human inbox to triage work.

demo commands 
python -m agent_hub serve --port 8080
python -m agent_hub dispatch
python -m agent_hub run-task-template sample-project summarize-input --input "review this idea"
python -m agent_hub run-pipeline sample-project sample-flow --input "demo"
python -m agent_hub list-human-inbox
python -m agent_hub dashboard

Fastest way to try it

Run the web surface, start the dispatcher, queue example work, then open the thin browser app. The repo includes an example project with one action, one task template, and one pipeline.

What you will see

A queue that can process tasks and pipelines, a saved-query workflow, a human inbox for triage, and a read-only dashboard fed from a single /dashboard snapshot.

What it is not trying to be

Not a hosted SaaS, not a multi-tenant workflow engine, and not a polished production UI. The point is to provide a small, extensible OSS control-plane baseline.
